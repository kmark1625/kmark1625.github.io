Ruby vs Javascript - Functions

What's so special about JavaScript functions anyways?

When first learning JavaScript, you may have noticed that functions look a lot like functions or methods in Ruby. The syntax for defining them seem to look very similar, in fact. Below is a simple example of a function in both Ruby and JavaScript that will add one to an input.


	# Ruby
	def addOne(num)
		return num + 1;
	end
					

	// JavaScript
	function addOne(num) {
		return num + 1;
	}

Functions in JavaScript are a little bit more special. They are what we refer to as first-class. Ruby's functions are not first-class (although there are procs which are). What makes first-class functions special is that they can be used as values, returned, and even passed into other functions. Below is a really interesting example of us using all three characteristics of functions (assigning a function to a variable, returning a function, passing a function to a variable). 


	// JavaScript
	function addOrSubtractN(n, operation) {
		if (operation === "add") {
			return function(num) { return num + n; }
		} else {
			return function(num) { return num -n; }
		}
	}
	var addOneFunction = addOrSubtract(1, "add");
	console.log(addOneFunction(5)); // Returns 6
